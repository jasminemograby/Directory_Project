# SendPulse - דרישת דומיין ארגוני

## מה הבעיה?

SendPulse **לא מאפשר** שימוש באימיילים חינמיים (Gmail, Yahoo, וכו') לשליחת אימיילים דרך SMTP API.

**הדרישה:** דומיין ארגוני (למשל: `yourcompany.com`)

---

## מה המצב עכשיו?

✅ **המערכת עובדת מצוין ב-Mock Mode!**

- כל הבדיקות עובדות
- כל הלוגים נשמרים
- כל הרשומות נשמרות ב-Database
- **רק האימיילים לא נשלחים אמיתיים** (עד שיהיה דומיין)

---

### אפשרות 2: דומיין חינמי (Zoho Mail)

**איך:**
1. **הירשמי ל-Zoho Mail** (חינמי)
2. **צרי דומיין חינמי** (למשל: `yourcompany.zoho.com`)
3. **או השתמשי בדומיין קיים** אם יש לך

**יתרונות:**
- חינמי
- עובד עם SendPulse

**חסרונות:**
- צריך להגדיר דומיין
- לוקח זמן

**אפשרויות:**

## מה קורה עכשיו?

### עם Mock Mode:

1. **HR מסיימת Step 3** עם רשימת עובדים
2. **המערכת בודקת אוטומטית** (Mock Mode - כל העובדים "לא רשומים")
3. **אם יש עובדים לא רשומים:**
   - המערכת מדפיסה לוג: `[Mock Data Fallback] SendPulse unavailable...`
   - נשמר רשומה ב-Database ב-`notifications` table
   - **HR לא מקבלת אימייל אמיתי** (עד שיהיה דומיין)

### מה HR רואה?

**עכשיו (Mock Mode):**
- HR **לא מקבלת** אימייל אמיתי
- אבל יש רשומה ב-Database שניתן לראות
- אפשר לראות את הרשומות דרך API או Dashboard (כשיהיה)

**כשיהיה דומיין:**
- HR תקבל אימייל אמיתי דרך SendPulse

## מה לעשות עכשיו?

### ✅ המשכי עם Mock Mode

1. **הכל כבר עובד!** ✅
2. **המערכת שומרת הכל ב-Database** ✅
3. **כשיהיה דומיין** → פשוט לעדכן את SendPulse

### 🔮 לעתיד (כשיהיה דומיין)

1. **השגי דומיין ארגוני** (או השתמשי בדומיין קיים)
2. **אמתי את הדומיין ב-SendPulse**
3. **עדכני את `SENDPULSE_FROM_EMAIL`** ב-`.env`
4. **הפעילי מחדש את השרת**
